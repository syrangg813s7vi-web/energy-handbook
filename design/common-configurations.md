# “常见组态”文章与动画设计

## 功能范围

维护一篇可持续扩展的“常见组态”文章，当前解释十一类新能源场站控制组态：

1. 并网点数据选择；
2. 光伏均衡；
3. 标幺值；
4. 过调检测；
5. 光伏储能变量分配；
6. 离网控制；
7. 并网控制；
8. `gridswitch_ctrl`；
9. 黑启动；
10. `fault_record`；
11. `sample_data_manager`。

每一节都回答“要解决什么问题、使用哪些输入、如何处理、输出到哪里、工程上应注意什么”，并提供可单步播放的交互动画。

## 分层与依赖

### 文章层

- 文件：`docs/knowledge/common-configurations.md`
- 职责：保存概念、控制边界、输入输出、公式、工程注意点和动画的文字等价内容。
- 公开合同：稳定的章节锚点和演示地址。
- 禁止依赖：不依赖浏览器脚本才能表达核心结论。

### 演示状态层

- 文件：`docs/.vitepress/theme/components/configurations/useStepDemo.ts`
- 职责：统一管理当前步骤、自动播放、复位和卸载清理。
- 公开合同：向演示组件返回当前步骤和三个控制动作。
- 禁止依赖：不读取文章 DOM，不了解任何具体组态数据。

### 演示表现层

- 目录：`docs/.vitepress/theme/components/configurations/`
- 职责：每个组态使用一个独立 Vue 组件，只保存本演示的教学状态、图形和文字。
- 公开合同：首屏 SSR 可读、键盘可操作、适配 320 px 以上宽度、尊重 `prefers-reduced-motion`。
- 禁止依赖：不读取文章 DOM，不调用外部 API，不依赖批阅层或服务端状态。

### 公共视觉与动效层

- 文件：`docs/.vitepress/theme/components/configurations/configuration-demo.css`
- 动效基准：采用 Codex 第一版本地演示的数据沿连线流动、活动通路点亮和数值连续插值；EnergyBook 既有卡片、按钮、主题色和背景保持不变。
- 公共合同：十一个演示共享既有控制按钮、指标行、当前步骤说明和进度条；具体图形与连续运动仍由独立组件负责。
- 连续运动：数据流粒子只使用 `transform` 与 `opacity`，状态变化优先使用合成友好的属性；不通过 JavaScript 逐帧改写布局。
- 节奏：步骤切换和连续数据流彼此独立，自动演示只负责切换教学状态，浏览器负责连续插值。
- 可访问性：在 `prefers-reduced-motion: reduce` 下停止粒子循环并将状态过渡缩短至近乎即时。
- 禁止依赖：不引入外部动效库，不使用滤镜模糊或大面积阴影，不创建常驻 JavaScript 动画循环。

### 装配层

- VitePress 主题入口只负责全局注册十一个语义化组件。
- Markdown 负责按章节组合组件，不掌握组件内部状态。
- 具体演示之间禁止相互导入；它们只单向依赖公共步骤状态层。

### 导航层

- 在 VitePress “工业控制”侧边栏增加“常见组态”。
- 不改变现有栏目和路由。

依赖方向为：

```text
导航 → 文章 → 独立演示组件 → 公共步骤状态层
```

状态层和演示组件不反向依赖文章或批阅层。

## 首版教学模型

### 并网点数据选择

演示单个物理并网点的主备测量选择：主测量正常、数据超时、保持最后可信值、切到备用、主测量恢复、延时回切。多物理并网点求和不在本动画范围内，但在正文中说明。

### 光伏均衡

演示三个可发能力不同的光伏单元：简单平均导致目标缺口、按能力同比例分配、单元可发能力下降、限幅、计算剩余裕度和二次分配。

### 标幺值

演示不同额定容量设备的 MW 实测值通过各自固定容量基准变成统一的 pu 值，控制器在 pu 尺度计算偏差，再将 pu 指令换回各设备的 MW 指令。动态可发能力用于均衡和限幅，不作为本动画的主控制基准。

### 过调检测

演示升功率过程：指令变化闭锁、正常跟踪、允许带内穿越、过调候选、持续超限确认和滞环复归。降功率采用对称判据，在正文中说明。

### 光伏储能变量分配

演示并网点有功目标在光伏和储能之间的场站级分配，采用储能正值表示放电、负值表示充电：

```text
P_POI* = P_PV* + P_BESS*
```

教学策略为“光伏优先、储能补差”，依次展示光伏独立完成目标、云影后储能补缺口、SOC 下限导致目标不可达、富余光伏给储能充电、SOC 上限导致限充与限光，以及光伏和储能协同升功率。动画必须明确区分“功率目标”“光伏可发上限”“储能功率上下限”和“SOC 能量边界”，不能把功率能力与能量状态混为同一变量。

实际工程还可能采用储能优先、计划曲线跟踪、SOC 恢复、削峰填谷、调频预留或电价优化策略；这些属于可配置的上层策略，不在首个教学动画中展开。

### 离网控制

演示并网运行、离网前功率预平衡、PCC 开关断开、构网型储能接管电压和频率、负荷阶跃、一次调节、二次恢复与能力不足时的负荷切除。教学边界是“带电母线上的并离网切换”，不与从无压状态开始的黑启动混淆。

### 并网控制

演示调度有功目标进入场站控制器，控制器读取并网点反馈，把增量分配给光伏与储能，并通过闭环使并网点有功、无功或电压跟踪目标。外部电网负责提供频率和相位参考，场站不把每台设备简单按同一 MW 或固定比例分配。

### `gridswitch_ctrl`

演示并网开关控制的状态机：接收合闸请求、检查保护和机构联锁、调节两侧电压/频率/相角、同期条件持续满足后生成许可、考虑开关动作时间发出合闸命令、核对辅助触点，以及保护动作时以更高优先级分闸并闭锁重合。

### 黑启动

演示全站无压起点下的恢复顺序：隔离故障、启动不依赖外部交流电源的构网型储能、软升压并励磁变压器、建立空载母线、分批恢复关键负荷、母线稳定后投入跟网型光伏，再逐级扩大普通负荷。动画必须突出励磁涌流、负荷阶跃和备用能力约束。

### `fault_record`

演示实时信号持续写入环形缓存、故障触发时冻结前触发数据、继续采集后触发窗口、生成带通道配置和统一时间戳的 COMTRADE 事件文件，再供保护行为、故障定位和模型回放分析。录波通道由组态选择，不宣称默认保存全场所有设备的全部电压电流。

### `sample_data_manager`

演示设备采样流经过点表识别、源时间戳与质量码检查、不同采样周期对齐、最新值和短时间窗缓存、原始与聚合数据分层存储，最终供实时算法、趋势和报表读取。该组件与 `fault_record` 分工明确：前者持续管理日常时间序列，后者在事件触发时保留高分辨率证据。

## 新增组态的扩展合同

后续每增加一种组态：

1. 在 `docs/.vitepress/theme/components/configurations/` 新增一个独立 Vue 演示组件；
2. 只依赖公共步骤状态层，不导入其他具体演示；
3. 在主题入口全局注册一个语义化组件名；
4. 在文章末尾新增一个二级章节；
5. 按“问题—输入—处理—输出—工程注意点”补齐正文；
6. 核心结论必须在 Markdown 中重复表达；
7. 运行构建、桌面与移动端交互检查。

公共状态层只包含已经稳定且完全相同的步骤控制逻辑，不包含领域数据或具体判据。公共视觉层可包含十一个演示一致的颜色映射、控件、间距和状态反馈，但不掌握具体图形布局。

## 验收标准

- 侧边栏能进入文章，生产路由为 `/knowledge/common-configurations`。
- 十一个演示都能自动播放、单步执行和复位。
- 动画在桌面与移动端无横向溢出，深浅色主题均可读。
- 曲线连线、流动粒子和状态插值与第一版本地演示保持同一动态语言，但不改变 EnergyBook 既有主题外观。
- 并网点数据选择和光伏均衡包含连续数据流；自动演示时状态切换没有明显跳帧。
- 支持 `prefers-reduced-motion`。
- 首屏有 SSR 输出；JavaScript 不可用时，正文仍能完整解释控制思想。
- `npm run check` 通过。
- 浏览器检查文章和十一个演示无控制台错误。

## 兼容、回滚与恢复

- 本功能只增加静态内容和主题组件注册，不修改批阅协议、服务端接口、依赖或构建流程。
- 回滚本次扩展时删除新增六个组件、撤销对应组件注册，并移除文章第六至十一节；既有五个组态和文章路由保持不变。
- 权威副本为 GitHub 仓库；演示不保存运行状态或外部数据，可从任意干净检出通过 `npm ci && npm run docs:build` 完整恢复。
- 本功能无数据库、上传文件、密钥或不可再生状态，RPO 由 Git 提交决定，RTO 为一次干净构建和 Pages 部署所需时间。
