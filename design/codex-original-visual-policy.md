# Codex 原版图形来源策略

## 目标

EnergyBook 文章中的图形与动画必须保持和 Codex 直接生成、经本地确认的版本一致，避免发布站点为了适配框架而重新绘制后出现节点、连线、布局、步骤或动效差异。

## 适用范围

本规则适用于文章正文中的流程图、结构图、控制逻辑图、状态图、静态插图和交互动画。新建或修改图形时立即执行；既有图形在后续维护时按同一规则迁移。

## 权威来源

- Codex 直接生成并经本地确认的 HTML、SVG、PNG、WebP 或其他仓库内原始文件，是视觉结构与表现的唯一权威来源。
- 原始文件必须进入 Git 管理的 `docs/public/` 或其他明确的版本化素材目录；开发机临时目录、缓存和构建产物不能作为唯一副本。
- 对交互动画，节点结构、路径、步骤数据、状态文字、动画时长和响应式重排均属于原版的一部分。

## EnergyBook 允许承担的职责

站点层只允许负责：

1. 文章中的语义化引用和标题；
2. iframe 或图片的加载、基路径和沙箱；
3. 深浅主题同步和响应式尺寸；
4. 可访问性标签、减弱动画偏好和文字降级；
5. 不改变原图的边框、留白与文章排版适配。

## 禁止事项

- 不得在 Vue、Markdown、CSS 或其他站点层重新绘制一个“看起来接近”的版本。
- 不得复制原版步骤数据后独立维护第二套状态机或动画时序。
- 不得为了适应文章宽度而修改原图内部节点关系；应通过原始文件自身的响应式设计或无损缩放解决。
- 不得只把截图作为交互动画的替代权威源；可使用截图做降级，但原始动画文件仍须版本化保存。

## 依赖方向

```text
文章 → 语义化嵌入层 → Codex 原版图形文件 → 原版所需的本地样式或数据
```

嵌入层不得反向掌握图形领域状态，也不得覆盖原版动效。

## 验收标准

- 发布页使用的文件与本地确认版本内容一致，不存在二次重绘副本。
- 普通阅读、专注阅读和移动端均无裁切、重叠或横向溢出。
- 深浅主题、交互控件、自动播放、单步和复位按原版工作。
- 原始文件和依赖均可从权威 Git 仓库在干净环境中恢复。
- 正文保留图形的关键结论，图形加载失败时仍可理解文章。

## 回滚与恢复

规则文档本身可独立回滚，不改变运行代码。具体图形迁移应以独立 PR 交付；如需回滚某张图，恢复其上一版原始文件或嵌入声明，不在站点层临时重画。
