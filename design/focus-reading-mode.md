# 专注阅读模式设计

## 功能范围

桌面端文章页增加一个“专注阅读”开关。开启后隐藏左侧章节导航和右侧本页目录，并把正文容器扩展到更宽的阅读区域；关闭后恢复 VitePress 默认布局。浏览器记住用户偏好，移动端继续使用现有的折叠菜单。

本功能不修改文章 Markdown、批阅任务、身份认证、动画或服务端接口。

## 分层与依赖

依赖方向保持为：

`VitePress 主题入口 → 阅读模式组件 → 浏览器状态`

`VitePress 主题入口 → 阅读模式样式`

各层职责如下：

- 浏览器状态：读取和保存布尔偏好，并在文档根元素上维护单一状态类。
- 交互组件：显示当前动作、处理点击，并提供键盘与辅助技术所需的状态。
- 主题样式：只根据根元素状态类调整 VitePress 布局。
- 主题入口：装配组件，不把阅读模式状态传入文章或批阅组件。

禁止依赖：

- 阅读模式组件不得读取或修改文章内容。
- 阅读模式不得调用批阅 API、身份接口或外部网络。
- 批阅组件不得依赖阅读模式状态。

## 状态与兼容

- 存储键：`energy-handbook:focus-reading`。
- 存储值：字符串 `true` 或 `false`。
- 本地存储不可用时，页面保持可操作，但偏好不跨刷新保存。
- 仅在宽度不小于 960 px 时显示开关并改变布局。
- 首页、404 页和自定义布局不显示开关。

## 验收标准

- 默认布局与现有 VitePress 行为一致。
- 点击“专注阅读”后，左右目录隐藏且正文宽度增加。
- 点击“显示目录”后恢复原布局。
- 刷新文章后保留当前偏好。
- 按钮可通过键盘操作，并暴露明确的 `aria-pressed` 状态。
- 移动端不显示开关，页面没有横向溢出。
- 现有测试和生产构建通过。

## 回滚

移除主题入口中的组件装配、`FocusReadingToggle.vue` 和对应的 `focus-reading` 样式即可完整回滚。文章内容和浏览器中的旧偏好值可以保留；没有组件读取时不会产生影响。
