# FBD 与 SFC：从“全网络扫描”到“活动步状态机”

在 IEC 61131-3 PLC 编程体系中，FBD（Function Block Diagram，功能块图）和 SFC（Sequential Function Chart，顺序功能图）都可以完成设备控制，但它们擅长表达的问题并不相同。

一句话概括：

- **FBD擅长表达“信号经过哪些逻辑得到输出”**；
- **SFC擅长表达“设备现在处于哪一步，满足什么条件后进入下一步”**。

为了把这种差别看得更直观，可以打开配套的 [PLC执行演示页面](../demos/plc/plc-fbd-sfc-execution.html)。页面使用同一条灌装线，同时展示两种程序的内部执行过程。

## 一、先认识这个灌装线案例

这是一个经过简化的**单工位瓶装液体灌装线**。空瓶由输送带送入加工位置，设备把瓶子固定后完成灌装和封盖，最后再把成品瓶送走。

整条生产过程可以概括为：

```text
空瓶进入 → 固定瓶子 → 注入液体 → 安装瓶盖 → 成品离开
```

案例的重点不是模拟一台完整的工业灌装机，而是用一个容易理解、具有明显先后顺序的工艺，说明PLC如何记住“当前做到哪一步”，以及如何根据现场信号进入下一步。

### 1. 设备由什么组成

这个案例包含四类主要执行机构：

| 执行机构 | PLC发出的控制请求 | 作用 |
|---|---|---|
| 输送带电机 | `输送带请求` | 把空瓶送入工位，或把成品瓶送走 |
| 夹紧机构 | `夹具请求` | 固定瓶子，防止灌装和封盖时移动 |
| 灌装阀 | `灌装阀请求` | 打开液体管路，向瓶内注入液体 |
| 封盖机构 | `封盖机请求` | 放置并压紧或旋紧瓶盖 |

这里的“请求”是PLC程序产生的逻辑输出。它不一定直接驱动电机或电磁阀，通常还会经过安全回路、驱动器或设备功能块，最后才形成物理输出。

例如：

```text
灌装阀请求 = TRUE
        ↓
检查自动模式、安全门、气压和设备故障
        ↓
条件全部允许后，物理灌装阀输出 = TRUE
```

因此，演示中的输出变量可以理解为上层流程对设备发出的控制命令。

### 2. PLC能看到哪些现场信号

为了判断每个动作是否完成，生产线上设置了五个简化的输入信号：

| 输入信号 | 可能的现场来源 | 表示的含义 |
|---|---|---|
| `到瓶` | 光电传感器 | 瓶子已经到达加工位置 |
| `夹紧到位` | 气缸磁性开关或限位开关 | 夹具已经完全闭合，瓶子固定完成 |
| `液位满` | 液位传感器、流量累计值或称重结果 | 本次灌装量已经达到设定值 |
| `封盖完成` | 封盖机完成信号 | 瓶盖已经安装完成 |
| `瓶离开` | 出口光电传感器 | 成品瓶已经离开加工位置 |

这些都是从设备进入PLC的输入信号。它们与 `X0～X4` 不同：

- `到瓶`、`液位满`等描述现场发生了什么；
- `X0～X4`描述PLC程序认为流程当前处于什么状态；
- `输送带请求`、`灌装阀请求`等描述PLC希望设备执行什么动作。

可以把三者理解成：

```text
输入信号：现场告诉PLC“动作完成了吗？”
内部状态：PLC记住“现在做到哪一步？”
输出请求：PLC告诉设备“现在应该做什么？”
```

### 3. 一只瓶子的完整加工过程

#### 第一步：等待空瓶

系统初始处于 `X0 等待` 状态。

- 输送带运行，把空瓶送向加工工位；
- 夹具、灌装阀和封盖机不动作；
- PLC持续观察入口光电传感器；
- 当 `到瓶=TRUE` 时，说明瓶子已经到达正确位置。

此时流程退出等待状态，进入夹紧状态。

```text
当前状态：X0 等待
设备动作：输送带运行
退出条件：到瓶 = TRUE
下一状态：X1 夹紧
```

在真实设备中，检测到瓶子后通常会先停止输送带。本教学模型将“停止输送带”体现为进入 `X1` 后不再产生输送带请求。

#### 第二步：夹紧瓶子

系统进入 `X1 夹紧` 状态。

- 输送带停止；
- PLC发出 `夹具请求=TRUE`；
- 气缸或电动夹具开始闭合；
- 当限位开关返回 `夹紧到位=TRUE` 时，表示瓶子已经固定。

流程随后进入灌装状态。

```text
当前状态：X1 夹紧
设备动作：夹具闭合
退出条件：夹紧到位 = TRUE
下一状态：X2 灌装
```

#### 第三步：灌装液体

系统进入 `X2 灌装` 状态。

- 夹具继续保持闭合，避免瓶子移动；
- PLC发出 `灌装阀请求=TRUE`；
- 液体通过管路进入瓶中；
- PLC等待 `液位满=TRUE`。

液位达到目标值后，流程退出灌装状态。进入下一状态后，灌装阀请求变为 `FALSE`，阀门关闭。

```text
当前状态：X2 灌装
设备动作：夹具保持，灌装阀打开
退出条件：液位满 = TRUE
下一状态：X3 封盖
```

实际设备可能不用液位开关，而是根据流量计、称重传感器或灌装定时器判断灌装完成。演示把这些复杂测量统一简化为 `液位满`。

#### 第四步：安装瓶盖

系统进入 `X3 封盖` 状态。

- 夹具继续保持闭合；
- 灌装阀关闭；
- PLC发出 `封盖机请求=TRUE`；
- 封盖机构完成取盖、放盖和压紧或旋紧动作；
- 封盖设备返回 `封盖完成=TRUE`。

流程随后进入放行状态。

```text
当前状态：X3 封盖
设备动作：夹具保持，封盖机动作
退出条件：封盖完成 = TRUE
下一状态：X4 放行
```

#### 第五步：放行成品瓶

系统进入 `X4 放行` 状态。

- 夹具松开；
- 输送带再次运行；
- 完成灌装和封盖的瓶子被送出加工工位；
- 出口传感器确认 `瓶离开=TRUE`。

瓶子离开后，流程回到 `X0 等待`，准备处理下一只空瓶。

```text
当前状态：X4 放行
设备动作：夹具松开，输送带运行
退出条件：瓶离开 = TRUE
下一状态：X0 等待
```

### 4. 五个工序的整体对照

| 当前工序 | 活动输出请求 | PLC等待的输入 | 条件成立后的下一工序 |
|---|---|---|---|
| X0 等待 | 输送带 | 到瓶 | X1 夹紧 |
| X1 夹紧 | 夹具 | 夹紧到位 | X2 灌装 |
| X2 灌装 | 夹具、灌装阀 | 液位满 | X3 封盖 |
| X3 封盖 | 夹具、封盖机 | 封盖完成 | X4 放行 |
| X4 放行 | 输送带 | 瓶离开 | X0 等待 |

这个循环会不断重复：

```text
X0 ─到瓶→ X1 ─夹紧到位→ X2 ─液位满→ X3 ─封盖完成→ X4 ─瓶离开→ X0
```

### 5. 演示做了哪些简化

真实灌装线还会处理更多情况，例如急停、安全门、气压不足、无瓶不灌装、缺盖、动作超时、人工模式、报警复位和废品剔除等。为了突出FBD和SFC的状态表达差异，演示作了以下简化：

- 同一时刻只加工一只瓶子；
- 五个现场完成信号由按钮依次触发；
- 所有设备动作都能正常完成，不模拟故障；
- `自动模式`和`安全允许`始终为 `TRUE`；
- 每个工序只有一个正常出口，不包含异常分支；
- 输送带、夹具、阀门和封盖机的底层控制被抽象为BOOL请求。

有了这幅现场图景，接下来再看FBD和SFC，就可以把每一行代码对应到具体的传感器、执行机构和工序切换。

## 二、两者都运行在PLC的周期任务中

理解FBD和SFC之前，首先要明确：它们通常都不是只运行一次的脚本，而是被PLC周期性执行。

一个典型PLC周期可以简化为：

1. 读取输入映像，例如“到瓶”“夹紧到位”“液位满”；
2. 执行用户程序；
3. 更新输出映像，例如输送带、夹具、灌装阀和封盖机请求；
4. 下一个周期重新开始。

演示页面中的按钮代表现场信号到来。例如单击“继续：发送‘到瓶’”，相当于在某次PLC扫描中让“到瓶”从 `FALSE` 变为 `TRUE`。

因此，FBD和SFC真正的区别并不是“一个循环执行、另一个不循环执行”，而是：**在同一个周期任务里，程序员如何组织状态、转换条件和动作。**

## 三、FBD如何表达这条灌装线

假设灌装线有五个工序：

```text
等待 → 夹紧 → 灌装 → 封盖 → 放行 → 等待
```

如果用FBD建立顺序控制，常见做法是由程序员自行定义一组BOOL状态变量：

```text
X0 = 等待状态
X1 = 夹紧状态
X2 = 灌装状态
X3 = 封盖状态
X4 = 放行状态
```

正常情况下只有一个状态变量为 `TRUE`。例如设备正在灌装时：

```text
X0=FALSE  X1=FALSE  X2=TRUE  X3=FALSE  X4=FALSE
```

程序还要显式计算每个状态的退出条件：

```text
T0 := X0 AND 到瓶;
T1 := X1 AND 夹紧到位;
T2 := X2 AND 液位满;
T3 := X3 AND 封盖完成;
T4 := X4 AND 瓶离开;
```

这里的 `T2` 并不只是“液位满”，而是“当前确实处于灌装状态，并且液位已经满”。这可以防止一个与当前状态无关的输入信号错误触发切换。

当 `T2=TRUE` 时，程序必须显式退出旧状态并进入新状态：

```text
IF T2 THEN X2 := FALSE; X3 := TRUE; END_IF;
```

其中：

- `X2 := FALSE` 表示退出“灌装”状态；
- `X3 := TRUE` 表示进入“封盖”状态。

“退出”不是PLC的一条独立物理输出，而是**表示当前状态的BOOL位不再活动**。类似地，`液位满`通常是输入信号，`X2`则是程序内部保存的状态。

最后，程序再由状态变量产生设备请求：

```text
输送带请求 := (X0 OR X4) AND RunOK;
夹具请求   := (X1 OR X2 OR X3) AND RunOK;
灌装阀请求 := X2 AND RunOK;
封盖机请求 := X3 AND RunOK;
```

这就是FBD版本的核心结构：

```text
输入信号 → 计算全部转换 → 更新状态BOOL → 计算全部输出
```

在演示页的“FBD周期执行”区域中，可以看到扫描指针依次经过 Network 1、Network 2 和 Network 3。即使当前只有 `T2` 可能成立，程序仍会按照编译后的执行顺序计算其他网络。蓝色表示正在执行，绿色表示本周期真正成立或产生影响。

## 四、SFC如何表达同一条灌装线

SFC把顺序控制中的三个核心概念直接变成语言结构：

- **STEP（步）**：设备所处的工序状态；
- **ACTION（动作）**：该步活动时需要执行的控制逻辑；
- **TRANSITION（转换）**：从当前步进入后继步所需满足的条件。

同一段灌装流程可以表达为：

```text
INITIAL_STEP X0_等待
    ACTION N: 输送带请求 := TRUE;
TRANSITION X0_等待 → X1_夹紧 WHEN 到瓶;

STEP X1_夹紧
    ACTION N: 夹具请求 := TRUE;
TRANSITION X1_夹紧 → X2_灌装 WHEN 夹紧到位;

STEP X2_灌装
    ACTION N: 夹具请求 := TRUE;
    ACTION N: 灌装阀请求 := TRUE;
TRANSITION X2_灌装 → X3_封盖 WHEN 液位满;
```

这里的 `TRANSITION X2_灌装 → X3_封盖 WHEN 液位满` 是一个完整的有向转换关系，包含：

```text
前置步 X2_灌装 + 转换条件 液位满 + 后继步 X3_封盖
```

它不是一个脱离活动步、独立运行的条件。当 `X2_灌装` 不是活动步时，这条转换并未被使能；即使“液位满”为 `TRUE`，它也不应把流程直接切到封盖。

## 五、PLC如何执行SFC

SFC运行时会维护活动步信息。演示页面用以下内部属性表示它：

```text
X0.X、X1.X、X2.X、X3.X、X4.X
```

其中 `.X` 表示该步当前是否活动；`.T` 通常表示该步已经活动的时间。例如：

```text
X2.X = TRUE
X2.T = 650 ms
```

表示“灌装步当前活动，并已持续约650毫秒”。这些名称和具体可用属性可能因开发环境而有差异，但“运行时保存活动步集合”是理解SFC的关键。

一次简化的SFC周期执行可以分为四个阶段：

1. 读取当前活动步集合；
2. 执行活动步关联的动作；
3. 检查从活动步出发、当前已使能的转换；
4. 若转换条件成立，停用前置步并激活后继步，然后保存新的活动步集合。

例如当前活动步是 `X2_灌装`：

```text
执行“夹具请求、灌装阀请求”
        ↓
检查“液位满”
        ↓
FALSE：继续保持X2
TRUE ：停用X2，激活X3
```

后继步 `X3` 被激活，并不一定表示它的ACTION已经在同一瞬间执行。动作何时首次执行、转换能否在同一扫描中连续跨越多个步，取决于具体PLC厂商的SFC执行语义和任务配置。演示页采用便于教学的模型：本周期末激活后继步，其动作在下一周期执行。

还要注意，SFC并不一定“每次只执行一个步”。顺序流程通常只有一个活动步，但在并行分支中可以同时存在多个活动步。更准确的说法是：**SFC每个周期处理当前活动步集合，而不是由程序员手写一组互斥BOOL位。**

## 六、状态表达是关键区别，但不是唯一差别

SFC直接表达状态机，确实是它和FBD最重要的差别。不过两者还存在几项关键区别。

| 比较项 | FBD | SFC |
|---|---|---|
| 主要表达对象 | 数据流、布尔逻辑、功能块关系 | 步骤、动作、转换和分支 |
| 状态保存 | 通常由程序员使用BOOL、锁存器或功能块显式实现 | 由SFC运行时维护活动步集合 |
| 顺序切换 | 显式计算条件，再RESET/SET状态 | 前置步、条件、后继步直接组成转换 |
| 周期执行观感 | 网络通常按确定顺序被扫描 | 围绕活动步执行动作并检查已使能转换 |
| 并行流程 | 需要自行组织状态位与互锁 | 可用选择分支、并行分支和汇合结构表达 |
| 调试观察 | 观察信号线、功能块和内部变量 | 直接观察活动步、转换条件和步持续时间 |
| 典型用途 | 联锁、模拟量处理、PID、设备功能逻辑 | 批处理、装配、输送、启停顺序、工艺流程 |
| 维护风险 | 顺序复杂时状态位和SET/RESET关系容易膨胀 | 流程直观，但步动作中的复杂算法仍需其他语言 |

## 七、为什么复杂顺序中SFC通常更简洁

在只有两三个状态时，FBD的BOOL状态机并不难理解。但随着工序增加，FBD通常还要增加：

- 每个状态的保存位；
- 每条转换的布尔逻辑；
- 旧状态RESET和新状态SET；
- 异常、暂停、复位和跳转逻辑；
- 防止多个状态同时为TRUE的互锁；
- 并行分支的同步条件。

如果流程有20个步骤，程序员不仅要画20个状态，还要维护状态之间的切换关系。SFC则把“步—转换—步”作为一等结构，程序图本身就是流程拓扑。在线调试时，活动步直接高亮，维护人员更容易回答三个问题：

```text
现在在哪一步？
这一动作为什么正在执行？
还在等待哪个转换条件？
```

这正是演示页中橙色活动步、相邻转换条件和PLC内部状态存储区想表达的内容。

## 八、如何选择FBD还是SFC

可以使用一个简单判断方法：

### 更适合FBD的场景

- 逻辑重点是输入、联锁、运算和输出之间的关系；
- 需要组合定时器、计数器、PID或设备功能块；
- 顺序状态很少，或者状态机不是主要复杂度来源；
- 团队已有成熟的FBD功能块库。

### 更适合SFC的场景

- 工艺天然可以描述为“先做A，条件满足后做B”；
- 步骤多，并且存在暂停、分支、并行或汇合；
- 调试时需要快速定位“流程卡在哪一步”；
- 工艺人员需要直接参与程序审查。

实际工程中，两者往往不是二选一。常见的组合是：

```text
SFC负责上层工艺顺序
        ↓
FBD功能块负责每台设备的联锁与控制
        ↓
物理输出
```

例如，SFC的“灌装”步发出 `灌装请求`，FBD中的灌装阀功能块再综合安全门、气压、手自动模式和故障状态，决定是否真正打开物理阀门。这样既保留了流程的可读性，也保留了设备控制的模块化和安全联锁。

## 结语

FBD和SFC都在解决PLC控制问题，但它们提供了不同的观察角度：

```text
FBD问：信号如何经过逻辑得到结果？
SFC问：现在处于哪一步，何时进入下一步？
```

在配套演示中，可以先观察FBD蓝色扫描指针如何走完整张网络，再观察SFC如何读取活动步、执行动作、检查相邻转换并保存新状态。两段动画控制的是同一台机器，却把“状态”放在了完全不同的位置：FBD由程序员显式搭建，SFC由语言结构和运行时共同维护。

这就是两者最本质、也最实用的区别。
